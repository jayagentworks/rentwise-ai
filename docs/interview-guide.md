# 面试讲解稿

## 30 秒版本

RentWise AI 是一个面向中国租客的决策 Agent，不做房源发布，而是在候选房源上计算真实总成本、多成员高德通勤、硬条件和偏好匹配。系统用 LangGraph 编排确定性流程，PostgreSQL checkpoint 支持失败恢复；LLM 只负责开放偏好映射和证据约束解释。合同模块采用视觉 OCR、确定性法律规则和带版本来源的解释，避免模型直接宣布违法。

## 架构问题

**为什么不是纯 LLM Agent？** 费用、路线和法律风险需要可复现证据。LangGraph 管流程，工具和规则管事实，LLM 只处理自然语言边界。

**如何保证高德稳定？** Redis 缓存相同地理编码和路线，全实例共享 Lua 限流器，将路线请求控制在每秒 3 次；失败返回 503，不偷偷降级为随机时间。

**多目的地如何排序？** 除加权平均，还计算最差通勤、每周家庭总时间和成员差距；任何成员超过独立上限都会触发硬条件冲突。

**记忆如何实现？** 浏览器保存匿名 ID 和一次性 Token，后端只保存 Token 哈希。偏好、收藏、历史和反馈存 PostgreSQL；用户可导出/导入 JSON 跨浏览器迁移。

**合同照片怎么办？** JPG/PNG/WebP 和扫描 PDF 先走视觉 OCR，再跑确定性规则；报告保存哈希和结论，不保存正文。法律来源记录适用地区、生效日和核验日。

**怎样评估？** Pytest 验证算法和 Skill，Playwright 验证浏览器流程，固定评估集验证硬条件、指标非负和不可验证偏好不进入评分。

## 可讨论的取舍

- 当前房源是模拟 Provider，所以“真实”仅指高德通勤，不声称房源实时有效。
- 匿名模式保护数据最小化，但清理浏览器会丢身份，因此提供档案导入导出。
- MinIO 默认关闭，只有明确授权才持久化文件。
